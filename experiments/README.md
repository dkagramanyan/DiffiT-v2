# Biased-generalization experiment for DiffiT

Reproduces Figure 1(a) of [Garnier-Brun et al., *Biased Generalization in Diffusion Models* (arXiv:2603.03469)](https://arxiv.org/abs/2603.03469) on the DiffiT architecture.

**What we're testing:** whether DiffiT, trained on two disjoint halves of ImageNet, exhibits a *biased generalization* phase — a span of training where the two models produce identical images under matched noise, even though the test loss is still decreasing.

**Plot produced:**
- X axis: kimg (images processed in thousands).
- Left Y (blue): sample-split cosine distance between images generated by model A and model B under identical noise trajectories.
- Right Y (red/orange): DSM test loss on the held-out val split, one curve per model.

The paper's signature is: the blue curve bottoms out *before* the red curve does. That's the biased-generalization window.

## Dataset split

Every launch uses the *same* deterministic partition (seed=42) of the dataset:

| Slice | Fraction | Purpose |
|---|---|---|
| val   | 0.10 | test-loss evaluation, shared across both models |
| trainA | 0.45 | training data for model A |
| trainB | 0.45 | training data for model B |

val, trainA, trainB are disjoint. Both A and B see the same val split, so `Loss/test` is directly comparable between the two runs.

## Step 1 — Train both models

Run the A sbatch and the B sbatch for your resolution of interest. They can run in parallel (two independent jobs) or sequentially. They use the same data archive; only the `--split` flag differs.

### 256² (recommended for the cheap variant)

```bash
sbatch experiments/sbatch/train_256_splitA.sbatch
sbatch experiments/sbatch/train_256_splitB.sbatch
```

**Config:** 2× H200, global batch 192, `--kimg=30000`, `--snap=10` → checkpoint + test-loss eval every 40 kimg.
**Walltime:** ~1–2 days per run.

### 512²

```bash
sbatch experiments/sbatch/train_512_splitA.sbatch
sbatch experiments/sbatch/train_512_splitB.sbatch
```

Global batch 128, `--kimg=20000`. Walltime ~3–4 days per run.

### 1024²

```bash
sbatch experiments/sbatch/train_1024_splitA.sbatch
sbatch experiments/sbatch/train_1024_splitB.sbatch
```

Global batch 32 (with `--grad-accum=2`), grad-checkpointing on, `--kimg=10000`. Walltime ~4 days per run.

### Manual launch (no sbatch)

Single GPU, 256²:

```bash
python experiments/train_sample_split.py \
    --outdir=./experiments/runs/256 \
```

## Step 2 — Analyze

Once both runs have produced several matched checkpoints, run the analysis:

```bash
python experiments/analyze_sample_split.py \
    --run-a=./experiments/runs/256/00000-diffit-256-splitA-batch192 \
    --run-b=./experiments/runs/256/00001-diffit-256-splitB-batch192 \
    --outdir=./experiments/analysis/256 \
    --num-samples=256 \
    --num-steps=50 \
    --batch-size=16 \
    --title="Biased generalization in DiffiT (256²)"
```

Or via the sbatch:

```bash
sbatch --export=ALL,RESOLUTION=256,\
RUN_A=/path/to/splitA,\
RUN_B=/path/to/splitB \
    experiments/sbatch/analyze.sbatch
```

**What the analysis script does:**
1. Finds matching `network-snapshot-<kimg>.pt` files in both run dirs.
2. For each matched pair, loads both EMA models onto one GPU.
3. Generates `--num-samples` images from each model using **identical noise trajectories** (same `z_T`, same per-step noise, same class labels).
4. Computes mean cosine distance between decoded images.
5. Reads `Loss/test` for each model from `stats.jsonl` (logged during training).
6. Writes a combined TensorBoard log to `--outdir` (`BiasedGen/cosine_distance`, `BiasedGen/test_loss_A`, `BiasedGen/test_loss_B`).
7. Saves `figure1a.png` (dual-axis plot) and `results.json` (raw data).

**Outputs:**
```
experiments/analysis/256/
├── figure1a.png
├── results.json
└── events.out.tfevents.*
```

## Step 3 — Interpret the plot

- **If the blue curve bottoms out clearly *before* the red curve**: biased-generalization phase confirmed in DiffiT. The gap between the two minima is your "biased window."
- **If the two minima coincide (or blue lags red)**: no biased-generalization phase under this training budget / architecture. May still appear with longer training or a smaller dataset.
- **If blue is monotonic down and never rebounds**: you haven't trained long enough to see the U-turn. Extend `--kimg`.

The paper observed the blue minimum at roughly the **epoch where generated images from A and B are nearly indistinguishable given matched noise**, long before overfitting (rising test loss) becomes visible.

## Gotchas and notes

- **Same `--seed`.** The training sbatches fix the RNG seed, so model A and model B start from identical weight initialization. This is important: the paper's effect is about training dynamics, not init variance.
- **Deterministic split.** The split seed (42) lives inside `train_sample_split.py`. Don't change it between the A and B runs.
- **Class-conditional.** Both models see all 1000 ImageNet classes; the split is over examples, not classes.
- **Matched noise is critical.** `analyze_sample_split.py` calls `torch.manual_seed(k)` right before each `p_sample_loop` to make sure both models consume the same `randn_like` sequence. If you change the sampler path, preserve this invariant.
- **No CFG during analysis.** The cosine-distance measurement uses plain sampling (no classifier-free guidance). CFG would amplify subtle differences and mask the phenomenon.
- **EMA vs. raw.** The analysis uses EMA weights (standard for diffusion eval). The training script saves both.
- **Memory for analysis.** `analyze_sample_split.py` loads both models onto one GPU. At DiffiT-XL scale (~561M params) that's fine on an H200; for 1024² the autocast activations during sampling dominate memory — the `analyze.sbatch` reduces `--num-samples` and `--batch-size` accordingly.
- **stats.jsonl is authoritative.** The analysis script reads test loss from `stats.jsonl` first; tfevents is only the fallback. If you move runs between machines, copy the `.jsonl` too.
- **Shrink the budget if needed.** The paper's U-turn on CelebA happens within ~1500 epochs of training. On ImageNet with DiffiT, the timescale is unclear. If 30k kimg is too long, try `--kimg=10000 --snap=5` first to see if the effect is visible earlier.

## Bringing down cost further

If two 30k-kimg DiffiT-XL runs is still too much, these knobs trade fidelity for walltime:

- **Smaller model.** Override `--model` with a custom constructor (add one to `diffit/diffit.py`) using e.g. `hidden=384, depth=14, num_heads=6, head_dim=64`. ~5× faster steps.
- **Smaller val.** `--val-batches=8` halves eval cost.
- **Sparser checkpoints.** `--snap=20` halves the number of analysis points, still enough to see the U-turn.
- **Fewer analysis samples.** `--num-samples=128 --num-steps=25` is enough for a qualitative figure.
- **ImageNet-100.** Rebuild the zip dataset with a subset of 100 classes. Much smaller, much faster, still class-conditional.
